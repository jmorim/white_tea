Exploratory metabolomic/longitudinal analysis of white tea

Tea is an important subject for me because it was the model crop on which my PhD thesis was based. At Tufts, we harvested tea grown in different seasons and elevations to see how its environment changed its flavor and health properties, all driven by its metabolism. Like many plants, tea uses chemical signals to communicate with its environment, since it can’t get up and walk away like an animal or even a microbe could. Evolution has driven myriad metabolic pathways to allow plants to defend themselves. This makes them particularly complex and challenging subjects of research, slowing studies on how crop qualities will change with the increasing threat of climate change.

For this demo/tutorial, I looked for data collected from Camellia sinensis samples on MetaboLights and found one study in progress by Wei and Li (https://www.ebi.ac.uk/metabolights/editor/MTBLS13336/overview). This study aimed to investigate the sensory and chemical metabolites, both endogenous to the tea and exogenous from microbial contributions, over long-term storage. We will focus on the metabolomics data herein.

This will take a while, as high resolution mass spec data can be several gigabytes per file, so if you’re playing along at home, you can execute this block and go get a coffee. In this case, they’re ~80 MB per file. I’ve never worked with Orbitrap data so that’s pretty neat compared to Q-TOF profile data.

Note that I had to replace the FILES/ prefix from the repo because I don’t need that in my local copy.

This didn’t work either due to timeouts with the ftp server. In the end, downloading the files sequentially with download_file was most reliable.

dir.create('_raw', showWarnings=FALSE)
if(length(study.info[, 'Raw Spectral Data File']) != length(dir('_raw'))){
  walk(study.info[, 'Raw Spectral Data File'], ~ download.file(
    url = paste0(mtbls_ftp_path(mtbls.repo), .x),
    destfile = paste0('_raw/', stringr::str_replace(.x, 'FILES/', ''))),
    method = 'libcurl'
  )
}

Data conversion

To read files with xcms, data need to be converted to .mzML format with MSConvert from the ProteoWizard package. Since I’m on Linux, I need a docker container to run this with a Windows image. On Windows, it can run natively. The code below detects the host environment and chooses the appropriate conversion command. The convertDtoMZML function from my LCMStools package only works for Agilent .d files. Docker will prompt for a password for sudo access. By using the Docker image, you agree to certain vendor licenses for converting proprietary formats to .mzML.

raw.files = dir('_raw', pattern = '\\.raw$', full.names = TRUE)

# Check if length mzML files matches length raw files
if(length(raw.files) > length(dir('_mzML', pattern = '\\.mzML$'))){
  if(.Platform$OS == 'Windows'){
    d.files |> walk(convertDtoMZML, path.out = '_mzML')
  } else if(.Platform$OS == 'unix'){
    walk(raw.files, ~ system(paste0(
      'sudo docker run -it --rm -v $PWD:/data proteowizard/pwiz-skyline-i-agree-to-the-vendor-licenses:latest wine msconvert /data/_raw/', basename(.), ' --filter "peakPicking true 1-" -o /data/_mzML --mzML'
      )
    ))
  }
}
